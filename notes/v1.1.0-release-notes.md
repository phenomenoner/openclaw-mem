# openclaw-mem v1.1.0

Release date: 2026-03-10

## Summary

v1.1.0 is a feature release focused on operator-safe memory workflows:

- deterministic topology query/drift tooling
- recommendation-first optimization checks (zero-write)
- expanded episodic capture (including conversation lanes) with stricter redaction defaults
- broader `openclaw-mem-engine` policy controls (docs cold lane, TODO guardrails, recall/capture receipts, embedding clamps)

Relative to v1.0.4, this release is largely additive (new commands + optional knobs). No mandatory migration step is required for baseline sidecar usage.

## Highlights

### 1) Deterministic graph/query plane

New operator-facing commands:

- `graph topology-refresh`
- `graph query ...` (`upstream`, `downstream`, `lineage`, `writers`, `subgraph`, `filter`, `receipts`, `provenance`, `drift`)

Why it matters: stable topology truth + rebuildable SQLite cache + drift/provenance reads gives operators a practical way to inspect relationships without mutating memory data.

### 2) Recommendation-only optimization observer

New command:

- `optimize review`

This reports staleness/duplication/bloat/connectivity/repeated-miss signals and emits recommendations only (no data mutation path in v0.1).

### 3) Episodic capture and ingest hardening

- capture lane expanded to conversation/tool/ops event families
- `episodes extract-sessions` + deterministic offset-state ingest flow
- stricter redaction posture (secret patterns + PII-lite + bounded payload defaults)
- retention defaults aligned for conversation event types

### 4) `openclaw-mem-engine` control surface expansion

- docs cold lane (`memory_docs_ingest`, `memory_docs_search`)
- TODO guardrails and bounded drop markers
- recall/capture policy + receipt tuning
- embedding clamp knobs for safer long-input behavior

## Upgrade / compatibility notes

- No explicit breaking schema migration identified in this cut.
- Existing sidecar installs can upgrade and continue with default behavior.
- If using `openclaw-mem-engine`, review and intentionally enable new knobs before production rollout:
  - `docsColdLane.*`
  - autoRecall / autoCapture / receipts controls
  - TODO guardrail knobs
  - embedding clamp controls (`embedding.maxChars`, `embedding.headChars`, optional `embedding.maxBytes`)
- If enabling episodic conversation capture, validate redaction + retention expectations in your environment before continuous follow mode.
