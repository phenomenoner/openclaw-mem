# 系列：記憶太用力

這組長文圍繞一個最近很容易被使用者感受到、但很少被產品說清楚的問題：

> LLM 不是只有忘太快，也會記得太用力。

起點是 Andrej Karpathy 對 personalization memory 的一句抱怨：模型常常因為太想證明自己記得你，反而把早就過期的訊號一再拖回對話。

這個資料夾收的是這次長文流程的核心產物：
- `positioning.md` — 讀者、主張、成功標準
- `sources.md` — 主要來源
- `outline.md` — 系列弧線
- `advisory_lane_status.md` — Claude / Gemini advisory lane receipts
- `gemini_review.md` — Gemini second-opinion 摘要
- `claude_review.md` — Claude second-opinion 摘要
- `part-1_problem-context.md`
- `part-2_solution-patterns.md`
- `part-3_openclaw-mem-practice.md`
- `part-4_open-questions.md`
- `claims.md` — 重要 claim 與來源對照
- `essence.md` — 載重想法濃縮
- `gtm_kit.md` — 可用的標題 / TL;DR / 社群文案

## 系列主軸
好的 LLM 記憶不是更會想起你，而是更知道什麼時候不該把想起來的東西說出口。
